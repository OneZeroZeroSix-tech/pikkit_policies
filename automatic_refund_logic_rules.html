Pikkit – Automatic Refund Logic Rules

Purpose:

To define automated decision-making logic for refunds and cancellations, reducing human error, fraud, and inconsistency.

1. AUTOMATIC FULL REFUND RULES

System issues full refunds where:

  • Courier never accepts

  • Platform billing error confirmed

  • Duplicate transaction detected

  • Unauthorized transaction verified

2. AUTOMATIC PARTIAL REFUND RULES

System issues partial refunds where:

  • Courier accepted but not arrived

  • Cancellation occurs within defined window

  • Platform-caused delay exceeds threshold

3. AUTOMATIC DENIAL RULES

System denies refunds where:

  • Pickup occurred

  • Delivery completed

  • Evidence missing

  • Policy violation detected

  • Dispute window expired

4. FRAUD PREVENTION LOGIC

Refund requests are automatically flagged where:

  • High refund frequency

  • Device/IP anomalies

  • Prior chargebacks

  • Misrepresentation patterns

Flagged accounts may be suspended.

5. FORCE MAJEURE LOGIC

No automatic refunds for:

  • Weather events

  • Natural disasters

  • Government actions

Manual review only.

6. FINALITY AND OVERRIDES

Automated decisions may be overridden only by authorized personnel.

All overrides are logged and auditable.

7. PROCESSOR ALIGNMENT STATEMENT

These rules are designed to align with:

  • Stripe dispute evidence standards

  • Visa and Mastercard reason codes

  • PayPal Seller Protection requirements
